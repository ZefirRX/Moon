# Техническое задание

## Проект
**Moon Messenger** — локальный клиент-серверный мессенджер для обмена текстовыми сообщениями в реальном времени.

Сноска символом * помечены задачи. Выполнение которых зависит от скорочти разработки и могут быть не реализованы

# 1. Общие сведения

## Наименование проекта

Moon Messenger

## Назначение

Разработка десктопного приложения для обмена сообщениями между пользователями локальной сети с использованием клиент-серверной архитектуры.

## Цель разработки

Создать демонстрационный мессенджер, позволяющий:

* регистрировать пользователей;
* выполнять авторизацию;
* обмениваться сообщениями в режиме реального времени;
* хранить историю переписки;
* отображать пользователей онлайн.

# 3. Архитектура системы

Система состоит из двух независимых приложений:

## 1. Сервер
Функции:
* запуск TCP-сервера;
* обработка подключений клиентов;
* маршрутизация сообщений;
* работа с базой данных;
* хранение пользователей;
* хранение истории сообщений;*
* отслеживание статуса пользователей.

## 2. Клиент
Функции:
* UDP-поиск сервера в локальной сети
* регистрация;
* авторизация;
* подключение к серверу;
* отображение списка пользователей;*
* обмен сообщениями;
* просмотр истории переписки.*


# 4. Функциональные требования

## 4.1 Регистрация пользователя
Пользователь должен иметь возможность:
* ввести логин;
* ввести пароль;
* подтвердить пароль;
* создать аккаунт.

Система должна:

* проверять уникальность логина;
* сохранять пользователя в БД;
* выводить сообщение об успешной регистрации.
## 4.2 Авторизация
Пользователь должен иметь возможность:
* ввести логин;
* ввести пароль;
* выполнить вход в систему.

Система должна:
* проверять данные через БД;
* разрешать вход только зарегистрированным пользователям;
* сохранять статус пользователя как «онлайн».*

## 4.3 Подключение к серверу
Клиент должен:

* подключаться по TCP;
* получать уведомление об успешном подключении.

Сервер должен:

* принимать подключения;
* хранить список активных клиентов.

## 4.4 История сообщений*

Система должна сохранять:
* текст сообщения;
* отправителя;
* получателя;
* дату отправки.

После повторного входа история должна автоматически загружаться из БД.

## 4.5 Уведомления

Система должна уведомлять пользователя о:
* успешном подключении;
* отключении сервера;
* новых сообщениях;
* ошибках авторизации.

# 6. Пользовательский интерфейс

## Окно авторизации
Элементы:
* поле логина;
* поле пароля;
* кнопка «Войти»;
* кнопка «Регистрация».

## Окно регистрации
Элементы:
* логин;
* пароль;
* подтверждение пароля;
* кнопка «Создать аккаунт».

## Главное окно мессенджера

Элементы:
* список пользователей;*
* область отображения сообщений;
* поле ввода сообщения;
* кнопка отправки;
* индикатор подключения.*

# 7. Нефункциональные требования

Система должна:
* поддерживать одновременное подключение не менее 10 клиентов;
* обеспечивать доставку сообщений в пределах локальной сети;
* сохранять данные после перезапуска сервера;*
* корректно обрабатывать потерю соединения.

# 8. Текущий статус реализации
На момент составления ТЗ реализовано:

* базовый TCP-сервер;
* подключение клиентов;
* рассылка сообщений между подключёнными клиентами;
* интерфейс окон авторизации;
* интерфейс окна чата;
* базовая клиент-серверная связь.

Требуют реализации:

* SQLite;
* регистрация пользователей;
* авторизация;
* список пользователей;*
* личные сообщения;*
* история переписки;*
* хранение статусов пользователей.*

# 9. Критерии приёмки

Проект считается завершённым, если:

1. Сервер успешно запускается.
2. Не менее двух клиентов подключаются одновременно.
3. Пользователь может зарегистрироваться.
4. Пользователь может войти в систему.
5. Сообщения доставляются адресату.
